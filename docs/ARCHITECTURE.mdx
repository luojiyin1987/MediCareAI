## 1. 概述 | Overview

### 1.1 目的 | Purpose

本文档描述了 MediCare_AI 系统的架构，这是一个结合 AI 智能诊断和全面患者管理的智能疾病管理平台。  
This document describes the architecture of the MediCare_AI system, an intelligent disease management platform combining AI-powered diagnosis with comprehensive patient management.

### 1.2 设计目标 | Design Goals

| 目标 / Goal | 描述 / Description | 中文描述 |
|------------|-------------------|----------|
| **可扩展性 / Scalability** | 处理不断增长的用户和数据 / Handle growing user base and data | 可扩展性 |
| **可靠性 / Reliability** | 通过适当监控实现 99.9% 正常运行时间 / 99.9% uptime with proper monitoring | 可靠性 |
| **安全性 / Security** | 符合 HIPAA 的数据保护 / HIPAA-compliant data protection | 安全性 |
| **性能 / Performance** | API 响应时间小于 2 秒 / < 2s API response time | 性能 |
| **可维护性 / Maintainability** | 代码整洁，文档完善 / Clean code with comprehensive docs | 可维护性 |

### 3.5 AI 服务层 | AI Service Layer

#### AI 诊断服务 | AI Diagnosis Service

**流程 / Flow:**
1. 接收患者症状和文档 / Receive patient symptoms and documents
2. 检索相关医疗指南 / Retrieve relevant medical guidelines (RAG)
3. 构建提示词 / Construct prompt
4. 调用 AI 模型 / Call AI model
5. 解析响应 / Parse response
6. 返回结构化诊断 / Return structured diagnosis

#### MinerU 文档提取 | MinerU Document Extraction

**功能 / Functions:**
- PDF 文本提取 / PDF text extraction
- 图片 OCR / Image OCR
- 表格识别 / Table recognition
- 结构化数据输出 / Structured data output

#### RAG (检索增强生成) | RAG (Retrieval-Augmented Generation)

**组件 / Components:**
- **文档向量化 / Document Vectorization**: Qwen API 生成嵌入 / Qwen API generates embeddings
- **向量存储 / Vector Storage**: PostgreSQL + pgvector
- **相似性搜索 / Similarity Search**: 余弦相似度 / Cosine similarity
- **上下文构建 / Context Building**: 动态检索相关指南 / Dynamically retrieve relevant guidelines

### 5.2 核心表结构 | Core Table Structures

#### users 表 / users Table
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    full_name VARCHAR(255) NOT NULL,
    role VARCHAR(50) NOT NULL DEFAULT 'patient',  -- patient, doctor, admin
    phone VARCHAR(50),
    is_active BOOLEAN DEFAULT true,
    is_verified BOOLEAN DEFAULT false,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE
);
```

#### medical_documents 表 / medical_documents Table
```sql
CREATE TABLE medical_documents (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    medical_case_id UUID REFERENCES medical_cases(id),
    filename VARCHAR(500) NOT NULL,
    file_type VARCHAR(50),
    file_size BIGINT,
    storage_type VARCHAR(50) DEFAULT 'local',  -- local, oss
    storage_path VARCHAR(1000),
    extracted_content TEXT,
    pii_cleaned_content TEXT,
    pii_cleaning_status VARCHAR(50) DEFAULT 'pending',
    upload_status VARCHAR(50) DEFAULT 'pending',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

#### vector_embeddings 表 / vector_embeddings Table
```sql
CREATE TABLE vector_embeddings (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    document_id UUID REFERENCES medical_documents(id),
    knowledge_base_id UUID,
    embedding vector(1536),  -- Qwen embedding dimension
    chunk_index INTEGER,
    chunk_text TEXT,
    metadata JSONB,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- 创建相似性搜索索引 / Create similarity search index
CREATE INDEX ON vector_embeddings 
USING ivfflat (embedding vector_cosine_ops);
```

## 6. 安全架构 | Security Architecture

### 6.1 认证与授权 | Authentication & Authorization

**JWT 认证流程 / JWT Authentication Flow:**
```
┌─────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────┐
│  Client │────▶│   Login     │────▶│   Verify    │────▶│ Generate│
│         │     │   Endpoint  │     │ Credentials │     │  JWT    │
└─────────┘     └─────────────┘     └─────────────┘     └────┬────┘
                                                              │
                                                              ▼
┌─────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────┐
│  Client │◀────│   Validate  │◀────│   Decode    │◀────│  JWT    │
│         │     │    JWT      │     │    JWT      │     │         │
└─────────┘     └─────────────┘     └─────────────┘     └─────────┘
```

**权限控制 / Permission Control:**
- 基于角色的访问控制 (RBAC) / Role-based access control (RBAC)
- 患者只能访问自己的数据 / Patients can only access their own data
- 医生只能查看被提及的病例 / Doctors can only view mentioned cases
- 管理员拥有完全访问权限 / Admins have full access

### 6.2 数据保护 | Data Protection

**PII 清洗 / PII Cleaning:**
- 自动检测姓名、身份证号、电话、地址 / Auto-detect names, IDs, phones, addresses
- 文档存储前清洗 / Clean before document storage
- AI 处理前脱敏 / De-identify before AI processing

**传输加密 / Transport Encryption:**
- HTTPS/TLS 1.3
- JWT 令牌签名 / JWT token signing
- API 请求加密 / API request encryption

### 6.3 审计日志 | Audit Logging

**记录内容 / Logged Content:**
- 用户登录/登出 / User login/logout
- 数据访问 / Data access
- 管理员操作 / Admin operations
- AI 诊断请求 / AI diagnosis requests

### 7.2 数据库扩展 | Database Scaling

**读写分离 / Read-Write Splitting:**
- 主库处理写操作 / Master handles writes
- 从库处理读操作 / Slaves handle reads
- 自动故障转移 / Automatic failover

**分区策略 / Partitioning Strategy:**
- 按时间分区 medical_cases 表 / Partition medical_cases by time

### 7.3 缓存策略 | Caching Strategy

**多级缓存 / Multi-level Caching:**
